## Зачем программисту знать алгоритмы

Алгоритмы — это не абстрактная теория, а практический инструмент, который нужен каждый день.

Основные причины, почему без них никуда:

1. Понимание алгоритмов помогает эффективно использовать любой язык программирования
2. Алгоритмические задачи развивают навык анализа и поиска решений
3. Умение писать оптимальный код ценится в команде и на собеседованиях

---

## Что такое сложность алгоритмов

Сложность описывает, сколько ресурсов требуется алгоритму при росте входных данных.

### Виды сложности

- **Временная сложность** — сколько операций выполняет алгоритм
- **Пространственная сложность** — сколько дополнительной памяти он использует

---


## Нотация Big O

Нотация Big O используется для оценки верхней границы сложности алгоритма.

Примеры:

- Один цикл по массиву — O(n)
- Два вложенных цикла — O(n²)
- Деление задачи пополам (например, бинарный поиск) — O(log n)

---

## Массивы: основные понятия

**Массив** — это упорядоченный набор однотипных элементов с фиксированной длиной.

Свойства массива:

- Все элементы одного типа
- Каждый элемент имеет индекс
- Доступ по индексу выполняется за O(1)

---


## Оценка операций с массивами

### Получение элемента по индексу
```java
public int getElement(int[] nums, int n) {
    return nums[n];
}
```

Временная сложность: O(1)
Память: O(1)
Практика: задачи и решения
Обмен значений без дополнительной памяти

### Поменять местами два числа, не используя дополнительную память.

Решение:

```java
public void swap(int a, int b) {
    a = a + b;
    b = a - b;
    a = a - b;
}
```
Прибавить единицу к числу в массиве

### Массив цифр представляет число. Нужно прибавить 1 и вернуть новый массив.

Примеры:
```java
[1,3,5,6] → [1,3,5,7]
[1,3,9,9] → [1,4,0,0]
[9,9] → [1,0,0]
```

Решение:
```java
public int[] plusOne(int[] digits) {
    for (int i = digits.length - 1; i >= 0; i--) {
        if (digits[i] != 9) {
            digits[i]++;
            return digits;
        }
        digits[i] = 0;
    }
    int[] result = new int[digits.length + 1];
    result[0] = 1;
    return result;
}
```
Время: O(n)
Память: O(n)
Поиск позиции вставки в отсортированном массиве


### Найти индекс, на который нужно вставить число, чтобы массив остался отсортированным.

Решение (бинарный поиск):

```java
public int searchInsert(int[] nums, int target) {
    int start = 0;
    int end = nums.length - 1;

    while (start <= end) {
        int mid = start + (end - start) / 2;
        if (nums[mid] == target) return mid;
        if (nums[mid] > target) end = mid - 1;
        else start = mid + 1;
    }
    return start;
}
```
Время: O(log n)
Память: O(1)
Перемещение нулей в конец массива


### Переместить все нули в конец, сохранив порядок остальных элементов.

Решение:

```java
public void moveZeroes(int[] nums) {
    int insertPos = 0;
    for (int num : nums) {
        if (num != 0) {
            nums[insertPos++] = num;
        }
    }
    while (insertPos < nums.length) {
        nums[insertPos++] = 0;
    }
}
```

Время: O(n)
Память: O(1)
Проверка массива на палиндром


### Определить, является ли массив палиндромом.

Решение:
```java
public boolean isPalindrome(int[] nums) {
    for (int i = 0, j = nums.length - 1; i < j; i++, j--) {
        if (nums[i] != nums[j]) {
            return false;
        }
    }
    return true;
}
```
Время: O(n)
Память: O(1)
Проверка массивов на анаграмму


### Понять, являются ли два массива анаграммами (цифры от 0 до 9).

Решение:
```java
public boolean isAnagram(int[] a, int[] b) {
    if (a.length != b.length) return false;

    int[] digits = new int[10];

    for (int num : a) digits[num]++;
    for (int num : b) digits[num]--;

    for (int count : digits) {
        if (count != 0) return false;
    }
    return true;
}
```
Время: O(n)
Память: O(1)
Анаграммы для строк

Решение для строк (латиница):
```java
public boolean isAnagram(String s, String t) {
    if (s.length() != t.length()) return false;

    int[] alphabet = new int[26];

    for (int i = 0; i < s.length(); i++) {
        alphabet[s.charAt(i) - 'a']++;
    }
    for (int i = 0; i < t.length(); i++) {
        alphabet[t.charAt(i) - 'a']--;
    }
    for (int count : alphabet) {
        if (count != 0) return false;
    }
    return true;
}
```
