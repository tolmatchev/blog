## Вводная задача

Частая ситуация в корпоративной среде — хочется быстро посмотреть на продукт, но инфраструктура полностью изолирована от интернета. Типовой запрос выглядит так:

- развернуть BI-инструмент в закрытом контуре;
- подключиться к внутренним базам данных;
- собрать несколько чартов для оценки возможностей продукта.

Для решения этой задачи достаточно виртуальной рабочей станции и инструментов контейнеризации: Docker и Docker Compose.

---

## Зачем нужен Docker

Docker — это средство контейнеризации, которое позволяет запускать приложения изолированно от операционной системы и ее окружения.

Ключевая проблема, которую решает Docker:  
как гарантированно запустить одно и то же приложение на любой платформе с одинаковыми настройками и результатом.

Контейнер включает в себя приложение и все его зависимости, благодаря чему запуск сводится фактически к «одному клику».

---

## Зачем нужен Docker Compose

Docker Compose используется, когда приложение состоит из нескольких компонентов, которые должны работать совместно. Типичный пример:

- веб-приложение;
- база данных;
- кеш или вспомогательные сервисы.

Docker Compose позволяет описать всю инфраструктуру в одном YAML-файле и запускать ее одной командой, вместо ручного управления каждым контейнером по отдельности.

---

## Подготовка окружения в закрытом контуре

Для работы в изолированной среде необходимо:

1. Установить Docker и Docker Compose с учетом ограничений закрытого контура.  
   В практическом примере использовались версии:
   - Docker 25.0.5  
   - Docker Compose 2.20.2

2. Подготовить контейнерные образы.
3. Перенести образы в закрытый контур.
4. Запустить все сервисы через Docker Compose.

После этого окружение готово к использованию.

---

## Как перенести образы в закрытый контур

Процесс передачи контейнеров состоит из нескольких шагов:

1. Загрузить образы под нужную архитектуру (например, x86_64):
```bash
   docker pull --platform linux/amd64 ghcr.io/datalens-tech/datalens-ui:0.2789.0
```
2. Сохранить образы в tar-файлы:
```bash
docker save -o image.tar ghcr.io/datalens-tech/datalens-ui:0.2789.0
```
3. Передать tar-файлы на сервер в закрытом контуре.
4. Загрузить образы на сервере:
```bash
docker load -i image.tar
```
5. Запустить инфраструктуру:
```bash
docker-compose up
```

## Итог

Даже в полностью закрытом контуре можно быстро развернуть и оценить open source продукт. Docker и Docker Compose позволяют заранее подготовить окружение, перенести его в изолированную среду и запустить без сложной ручной настройки. Это делает прототипирование быстрым, воспроизводимым и удобным даже при жестких инфраструктурных ограничениях.
